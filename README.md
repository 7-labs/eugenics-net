# Eugenics History & Bioethics Archive

Astro static content site for `eugenics.net`, built around a critical education/archive strategy.

## Runtime Boundary

This project lives on the protected SSD workspace. Local work is limited to source edits, docs, git/search, and short non-runtime inspection.

Do not run dependency installation, dev servers, previews, Playwright, Docker, databases, or watchers from this SSD directory. Use OpenClaw for npm/build/preview/browser QA.

## OpenClaw Commands

The OpenClaw workspace is:

```text
/Users/openclaw/test-workspace/eugenics-net
```

Expected validation flow:

```bash
~/.codex/bin/openclaw-ops health
bash ./deploy.sh preflight
bash ./deploy.sh readiness
bash ./deploy.sh validate
```

After OpenClaw build/export/check passes, sync only the validated generated root output and lockfile changes back to the SSD project. Do not run `npm install`, `npm run build`, `npm run preview`, or Playwright from this SSD directory.

The browser QA script reads `sitemap.xml` and captures desktop and mobile screenshots for every generated route.

`bash ./deploy.sh deploy-closeout` is the production acceptance gate after Cloudflare auth, deploy, and custom-domain binding are fixed. It checks Cloudflare status, local production readiness, local production smoke, OpenClaw production readiness, and OpenClaw production smoke.

The root deployment wrapper is:

```bash
bash ./deploy.sh preflight
bash ./deploy.sh readiness
bash ./deploy.sh validate
bash ./deploy.sh deploy-cf
bash ./deploy.sh deploy-closeout
bash ./deploy.sh production-smoke
bash ./deploy.sh cf-status
```

Current production blockers as of 2026-06-12: `https://eugenics.net/` redirects to Dynadot's parked-domain flow, not this Astro site, and OpenClaw has no non-interactive Cloudflare token. Cloudflare Pages deploy, custom-domain/DNS binding, and Cloudflare status/log evidence cannot close until those are fixed.

## Source Structure

- `src/pages/` - public static routes and sitemap endpoint
- `src/content/articles/` - foundation article collection
- `src/layouts/` - shared page and article layouts
- `src/components/` - navigation, source lists, warnings, archive template
- `src/data/` - site navigation, glossary terms, archive categories
- `src/styles/global.css` - global visual system
- `scripts/openclaw-browser-qa.mjs` - OpenClaw Playwright QA
- `scripts/site-integrity.mjs` - generated-route, metadata, sitemap, and link gate
- `scripts/production-smoke.mjs` - production URL, parked-domain, content, and security-header smoke
- `scripts/cloudflare-readiness.mjs` - Cloudflare credential, DNS, and parked-domain preflight
- `scripts/export-root.mjs` - copies OpenClaw `dist/` output to project root
- `scripts/sync-openclaw.sh` - rsync helper for the OpenClaw workspace
- `deploy.sh` - local orchestration wrapper for OpenClaw validation, Cloudflare Pages deploy, and production smoke
- `public/robots.txt` - crawl policy copied into build output
- `public/_headers` - Cloudflare Pages security headers and cache policy
- `public/_redirects` - Cloudflare Pages redirect policy

## Public Surface

- Core pages: Home, History, Bioethics, Teaching, Archive, Glossary, Policy, About, Content Warning
- Corrections route: `corrections.html`, using `corrections@eugenics.net`
- Updates route: `updates.html`, recording publication-state notes, corrections, and production blockers
- Foundation articles: 25 source-backed English pages generated at top-level `.html` URLs
- `sitemap.xml` and `rss.xml` are generated by Astro at build time
- `llms.txt` records the project stance and flagship routes for AI/search summarization
- `404.html`, favicon assets, and `site.webmanifest` are generated into the static root output

## Pre-Public Launch Checklist

- Provision and monitor `corrections@eugenics.net`.
- Keep `docs/LEGACY-URLS.md` as the source of truth for old URL redirect or 404 decisions.
- Add expert/community review before publishing any raw or annotated primary source items.
- Deploy to Cloudflare Pages and bind `eugenics.net` as a custom apex domain.
- Confirm production security headers, HTTPS/HSTS, deployment target, and non-Dynadot production response.
- Verify Google Search Console and Bing Webmaster access after launch.
