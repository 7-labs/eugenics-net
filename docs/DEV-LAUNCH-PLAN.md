## 0. Non-Negotiable Guardrails (READ FIRST — anti-drift)

These rules override any "improvement idea" the implementation agent has. Violating any of them means the task is wrong even if the code works.

### 0.1 Scope guardrails

- **This is a static Astro site. It stays a static Astro site.** Do NOT add: a backend server, database, CMS, comment system, user accounts, contact forms, API routes, SSR/hybrid output, React/Vue/Svelte islands, Tailwind, or any CSS framework. `output: "static"` in `astro.config.mjs` is permanent.
- **Do not change the URL scheme.** `build.format: "file"` produces root-level `*.html` URLs (e.g. `/what-is-eugenics.html`). All existing URLs must keep working byte-identically in the sitemap. No trailing-slash migration, no `/articles/` prefix.
- **Dependency budget.** Allowed additions in this plan, and ONLY these: `@astrojs/rss`, `pagefind`. Anything else requires explicit human approval first. Never run `npm audit fix --force`.
- **Editorial red lines (from CONTENT-SOP):** the site documents and critiques eugenics; it never endorses it. No ads, no affiliate links, no DNA-testing lead generation, no medical/reproductive/legal advice, no comments, no user uploads, no unreviewed primary-source downloads. Every public page keeps its source packet. Do not weaken the global alert banner, content warnings, anti-endorsement positions, or the no-advice gate.
- **Do not delete or rewrite existing articles' substantive content.** Content tasks in Phase 4 are additive (new articles, new packets) or mechanical (linking, metadata). Substantive rewrites of historical claims require the human review gate.

### 0.2 Runtime boundary (environment)

- Local Mac/SSD checkout = source edits, git, docs, short static checks (`node scripts/*.mjs`) only.
- ALL runtime work — `npm ci`, `npm run build`, preview, Playwright/browser QA, Lighthouse, wrangler deploy — runs on OpenClaw via the existing wrapper: `bash ./deploy.sh validate` etc. Never run dev servers or installs from the SSD checkout.
- Cloudflare credentials: never print token values. Presence-check only (the scripts already do this).

### 0.3 Evidence rule

A task is "done" only when its **Acceptance** block passes and the command output is captured in the closeout. "Build passed" is not acceptance for UI/SEO tasks; browser QA evidence is.

### 0.4 Task protocol

- Work the phases in order. Within a phase, tasks marked ⚙ are agent-executable; tasks marked 👤 are **USER ACTION REQUIRED** (registrar/dashboard/credentials) — do not attempt them, do not fake them, just verify their results when told they are done.
- One commit per task ID, message prefixed with the task ID (e.g. `P1-3: add favicon set`).
- After each phase: run `bash ./deploy.sh validate` (OpenClaw full gate) and record results in `ops/deploy-ledger.jsonl` + update `PROJECT_STATE.md`.

Fill `sourceCoverage` (role/claims/limits/sensitive-language/affected-communities), `claimMap`, `deepDiveSections`, `teachingUse`, `doesNotDo`.
2. Then write `src/content/articles/<slug>.md` matching the existing frontmatter schema exactly (look at `eugenics-in-sweden.md` as the canonical standard-tier example): summary, position, contentWarning, discussionQuestions, faqs, related, learningObjectives.
3. Add to `related` arrays of 2–3 existing sibling articles; add to relevant hub page lists; sitemap picks it up automatically.
4. `npm run check:site` on OpenClaw must pass (word count, packet linkage, claim-map label integrity gates all enforced automatically).
5. Mark `reviewStatus` as pending human subject-matter review — do not claim review completed.

Acceptance per article: `check:site` green; renders in browser QA; human owner signs off on the diff before deploy (content goes out in weekly batches, not per-article deploys).

## 7. Deliberately Out of Scope (do not build, even if it seems helpful)

- Backend/API/database/CMS/auth/comments/newsletters/forms (static + email only, per SOP).
- i18n/translations (revisit only after English cluster is established).
- Dark mode (low value for print-oriented archival audience; CSS budget better spent on print styles).
- Per-article generated OG images, webfonts, image CDNs, service workers/PWA offline.
- Paid anything: ads, affiliates, sponsorships are permanently forbidden by the editorial policy.
- CI/CD pipelines replacing `deploy.sh` (GitHub Actions has no OpenClaw/Cloudflare creds; deploy stays operator-run).
- Donating/funding widgets — needs a human legal/policy decision first.
